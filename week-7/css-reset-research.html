CSS Reset & Normalize

Introduction to CSS Reset & Normalize

CSS resets and normalizations are two different tools used for removing and reformatting default browser styles in your CSS style sheet.

While both tools have different purposes and methods, they aim to create a consistent baseline for styling across different browsers.

This page will dive into some basics around CSS resets and normalize.css.

Why Reset?

The primary reasons to reset your CSS styles are:

  • Browser Presentation: All browsers have different styling 'defaults' that define how they handle CSS content like lists, margins, padding, headings, etc.
  • Universal Selector Failures: Using a universal selector to set all of the margins and padding of your style sheet to 0 can lead to multiple browsers interpreting spacing differently.
  • Mass Removal of Styles: Certain browser set default styles for multiple elements that can be a pain to manually remove and interfere with design.

Resetting your CSS styles will help create a consistent baseline across browsers, strengthing and refining your web design.

Josh vs. Eric

Eric Meyer's outdated CSS reset and Josh Comeau's more modern CSS reset approaches vary in several ways:

  • Browser and Default Style Resets: While Eric chooses to reset all browser default styles, Josh takes the opposite approach arguing that modern browsers already have fairly consistent layout and spacing.
  • Specificity: Josh's modern CSS reset chooses to be more selective about exactly what NEEDS to be reset in comparison to the broader nature of Eric's CSS reset.
  • Length and Size: Comparing the two resets, Josh's is far more compact and efficent compared to the lengthy size of Eric's reset.

Overall, Josh Comeau's approach to the CSS reset is far more useful in a modern context compared to the outdated, lengthy reset created by Eric Meyer.

Using Reset and Normalize

Resetting or normalizing CSS styles requires a basic structure comprised of:

  • Selection: First, select the content in which you want to apply the reset or normalization using css selectors.
  • Deciding Parameters: Decide how agressive you want your reset or normalization to be by determining which specific styles you will target.
  • Setting Styles: Apply the desired reset or normalization to the selected styles by placing it within the selector tags.

This is the basic skeleton on which you can build a variety of different normalizations or resets on.

Boiler Plates

CSS boiler plates set up a generic structure for your style sheet that have some advantages and disadvantages.:

Advantages:

  • Fast and easy to implement (often copy-paste)
  • Consistent across web projects
  • Often include CSS resets/normalizations natively

Disadvantages:

  • Can be overly generic or not specific to your needs
  • Less customization control for the basis of your website
  • Can be hard to understand for beginners and therefore difficult to edit

Knowing the pros and cons to boiler plates helps you decide when and how to use them effectively.

Normalize.css vs. Resets

Normalize.css and CSS resets differ in many ways:

  • Specificity: While normalize.css focuses on what specifically needs to be reset, a CSS reset generally removes all built-in browser styling.
  • Length: While a normalize.css is typically condensed and short in length, a CSS reset tends to be longer and more comprhensive as it applys to more styles.
  • Aggressiveness: Normalize.css is less aggressive, making minimal changes to preserve useful defaults, whereas CSS resets are more aggressive, removing most or all default styling.

Whether or not to use a normalize.css or a CSS reset depends on your project's specific needs and goals.
